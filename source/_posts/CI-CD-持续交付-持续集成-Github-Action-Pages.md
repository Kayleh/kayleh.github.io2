## 持续集成 、交付、部署

持续集成是指，频繁地（一天多次）将代码集成到主干。 每次集成都通过自动化的构建（包括编译，发布，自动化测试）来验证，从而尽早地发现集成错误。许多团队发现这种方法可以显著减少集成问题，并允许团队更快地开发高质量的软件。 这是持续交付和持续部署的基础。

## 使用Github Action

GitHub Actions 是一个强大的持续集成/持续交付（CI/CD）工具，它可以与 GitHub 存储库集成，帮助自动化软件开发工作流程。
